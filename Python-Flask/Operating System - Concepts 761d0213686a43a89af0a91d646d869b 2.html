Program - Process and States of Process

Program Vs Process :

Consider an example where you have to open the google chrome.

what will happen -:

  • The CPU got triggered to open google chrome.
  • The CPU will look for chrome in the RAM
  • If it finds then show that if not then search in the hard-drive
  • In the hard drive it will find the program and make a copy of it which is called a process
  • And Hence, we can open two windows of chrome itself which are nothing but two process of chrome running that is two copies of chrome moved to RAM from harddisk

States of a process :

  • New State : When the program is making the process
  • Ready State : Instance or process is generated and ready for the CPU to take action.
  • Running State : Process is running by the CPU
  • I/O State : Process is ready for some I/O.
  • Terminated State : Process has done execution and has been removed from the RAM.

Degree of Multiprogramming

Maximum number of processes which are present inside our computer is called degree of multiprogramming.

For example

if XPLANE takes 1 GB of RAM per process and total free RAM in user space is 4 GB then the degree of multiprogramming is 4.

Types of Operating Systems

Basic Terminologies :

  • CPU efficiency : CPU efficiency is somewhat related that for how less time the CPU was not working, for example in 10 seconds, the CPU needed to be idle for 2 seconds and working for 8 seconds then that CPU is 80% efficient.
  • Parallel Processing : Parallel processing is a type of processing when the processes are executed by the CPU at the same time. For example, a mother is feeding her two children with both her hands, left for one children and right for other.
  • Concurrent Processing : Concurrent processing is a type of processing when the CPU is focusing on one process at one time , For example, mother uses only one hand to feed her two children, when one child is chewing she use the hand to feed the other child and vice-versa.

Types of Operating Systems :

  • Batch Operating Systems : Batch operating systems have two properties, they only have one CPU and they can only hold one process in their RAM on a time point, that means a process need to be terminated if you want to open another process.

    They have Degree of Multiprogramming - 1

    • Disadvantages :
      • Very Low CPU efficiency : Consider an example where you are using batch operating system for typing your name.
        • You type something on Keyboard ( I/O process, CPU idle).
        • CPU understands that ( executing ) ( CPU running )
        • Output on display ( CPU idle, I/O process )

        As you know that execution and I/O can't be done at the same time, you can see how many times CPU goes idle ( resting CPU , not working) and that make it super inefficient .

  • Multiprogramming Operating Systems : This types of systems are widely used in today, they have one CPU and they can store more than one process in RAM and that makes it cost effective as well as they provide average efficiency for use. They use concurrent processing.

  • Multiprocessing Operating System : This type of processing systems have more than one CPU and they can also handle one than one process in their RAM. Like you might have heard octacore CPU that actually means that there are 8 processors embedded in the CPU which can undergo the processes that actually means Multiprocessing OS.
    • The major disadvantage they bring to us is their high cost.

Process Control Block and Attributes of a process :

Terminologies :

  • Active Entity and Passive Entity : When you load up a process then there are two thing in the hard - disk before going to the RAM, one is the part which would be pushed to the RAM while there would be other part which is the main program, always in the Hard-disk, this part is called the passive entity while the other part is called the active entity.

Process Control Block :

When the process is load up there is not only the copy of the program that is placed inside the RAM, in RAM their is a whole process control block or PCB which is placed.

In a PCB there are 4 parts basically, Stack Memory, Heap Memory, Data Part and Program Part

  • Stack Memory : A stack memory stores all the functions of the program which will be executed at some point of time.
  • Heap Memory : Heap Memory stores all the things which are dynamically allocated, which means the allocation was not done on build-time but it was done on the runtime.
  • Data Part : The variables which are attached to a function will go with the functions in the stack memory there is no issue with that but the variable which are not associated with any functions like global variables and static variables, where will they go?, they will be stored in data part of the PCB.
  • Program-Part : This is where the program itself get stored.

For several processes there are several PCB's controlling the process environment.

Process Attributes :

Scenario to be understood : So till now it has been made clear that how CPU works, it is nothing but a resource manager somehow managing the processes's PCBs, this work is somhow done by schedulers, a process is scheduled means that processor is currently executing that operation and a process is truncated that means the process is eliminated by for sometime. Now, there are various scheduling algorithms which we use for scheduling processes, like first come first serve or shortest job first algorithm, now to understand process attribute, let's take an example that a processor is using a shortest job first scheduling algorithm. Now take two processes, A and process B, A take 5 ns to execute while B takes 2 ns to execute, the scheduling algorithm used the algo and obviously picked up process B to implement, B has somewhat 100 lines of code and 100 files to be read by B to be executed, imagine any game in your mind, now the user pushed another PCB of process C which takes only 1 ns to execute, now according to the algo B should not be executed first, now C is the top priority. Now, C would get what it deserve, the first priority, say B has been executed till line number 20 okay and read 10 files out of 100 and also has opened 5 files out of 10 ( maybe displayed (I/0)) understand it like you are playing CSGO and a gun is loaded but the player model is not loaded yet. So, B need to be forcefully stop now and something we should also think with our common sense, 20 lines of code has been already executed, when we will come back to execute process B after we execute C we will not load those 20 lines of code, that will be efficient, 10 files we have already read so we will not read them again, 5 files have already displayed, we will not display them again, we need everything to be effiecient na, why we will do the work that we have already did. So, we need to store some information about B that how much work has been done to B first by the CPU, so that when we come back to B after the execution of C we have a clear knowledge of what we have to do next and we won't repeat things and are super efficient. That's why we use process attributes, they are nothing but some attributes which help us to store the information about a particular process.

  • Process ID : Process ID is assigned to every process to every process automatically so that it can be easily identified by the CPU.
  • Process Counters : Process counters gives us information that how much process has been executed, in our case that is line 20.
  • Process State : Truncated or Scheduled.
  • General Purpose Registers : Registers are also memory blocks like heaps and stack, so consider that till the 20th line has executed the program has stored some information, consider playing CSGO, the player has 20 gold coins, so now we are terminating B we need to store this information also somewhere, so we store such information in registers.
  • Priority : The priority of the process according to the scheduling algorithm .
  • List of open files : The list of files B have read till now so you don't have to read them again
  • List of open devices : List of open devices for the process, take example of headphone support while playing CSGO, the process must remember na that it has given headphone support already so don't need to give them or verify them back.
  • Protection : While C is executing, B also has something in the RAM, so it is ensured that C do not mess up with the registers of B, this is reffered as protection.
